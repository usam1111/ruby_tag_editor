# ルビタグ編集ツール 設計書

## 1. アプリケーション構成概要
本ツールはReactを用いて構築するシングルページアプリケーション（SPA）です。
画面遷移はなく、1つのメインコンポーネント内で3つの主要なUI部品（ソーステキストエリア、プレビューエリア、ボタンエリア）を配置します。

---

## 2. コンポーネント設計

### 2.1. App (メインコンポーネント)
全ての状態（State）とイベントハンドラーを管理し、子コンポーネントへPropsとして渡します。

### 2.2. SourceEditor (ソーステキストエリア)
- **役割**: ユーザーが直接テキストを編集・選択するUI。
- **UI要素**: `<textarea>`
- **受け取るProps**:
  - `text` (string): 現在のソーステキスト
  - `onChange` (function): テキスト変更時のイベント
  - `onMouseDown` (function): マウスダウン時のイベント（初期テキスト消去判定用）
  - `onMouseUp` (function): マウスアップ時のイベント（テキスト選択・タグ挿入用）

### 2.3. Previewer (プレビューエリア)
- **役割**: ソーステキスト内の`<ruby>`タグを解釈し、実際のルビ付きテキストとして描画するUI。
- **UI要素**: `<div>` (読み取り専用)
- **受け取るProps**:
  - `text` (string): 現在のソーステキスト

### 2.4. ButtonGroup (ボタンエリア)
- **役割**: コピー、ひとつ戻る、クリアの各アクションを実行するUI。
- **UI要素**: `<button>` × 3
- **受け取るProps**:
  - `onCopy` (function): ソースをコピーするイベント
  - `onUndo` (function): ひとつ戻るイベント
  - `onClear` (function): クリアするイベント
  - `canUndo` (boolean): 戻る履歴があるかどうかのフラグ表示用

---

## 3. 状態管理 (State)

メインコンポーネント(`App`)にて以下のStateを管理します。

| State名 | 型 | 初期値 | 役割 |
|---|---|---|---|
| `text` | string | "ここに<ruby=もじ>文字</ruby>をいれてください" | 現在表示されているソーステキスト |
| `history` | string[] | `[初期テキスト]` | 履歴を管理する配列。（最大6件保存。現在値＋5回前） |
| `isInitialText` | boolean | `true` | 現在のテキストが初期テキストであるか判定用フラグ |

---

## 4. ロジック設計 (イベントハンドラ)

### 4.1. 初期テキストの消去 (`handleMouseDown`)
- `isInitialText` が `true` の場合、マウスダウン時に `text` を空文字列にし、`isInitialText` を `false` に更新する。

### 4.2. ルビタグの挿入 (`handleMouseUp`)
- `textarea` の `selectionStart` と `selectionEnd` を取得する。
- 選択範囲が存在する場合（`selectionStart !== selectionEnd`）のみ処理を行う。
- **処理手順**:
  1. `<textarea>`からフォーカスを外さないよう制御しつつ、選択されていた文字の前後を分割する。
  2. 新しいテキストを `前半部分 + "<ruby=>" + 選択文字 + "</ruby>" + 後半部分` として結合。
  3. `<textarea>` のカーソル位置を、挿入した `<ruby=` の直後（右側）に再設定する。（DOMのRefを用いて `setSelectionRange` を実行）
  4. 選択状態を解除する。
  5. 状態（`text`）を更新し、履歴（`history`）に保存する。

### 4.3. リアルタイム更新 (`handleChange`)
- テキストエリアがキーボード等で直接編集された際の発火イベント。
- 変更後のテキストを `text` に反映し、プレビューエリアを再レンダリングする。
- 同時に入力確定のタイミング等で履歴（`history`）に追加する。

### 4.4. クリップボードへのコピー (`handleCopy`)
- `navigator.clipboard.writeText(text)` を使用して、現在のテキストをクリップボードにコピーする。

### 4.5. 履歴の管理と「ひとつ戻る」 (`handleUndo`)
- **履歴保存ルール**: `text` に変更が加わった際、`history` 配列の末尾に新しいテキストを `push` する。配列長が6を超える場合は先頭を `shift` で削除し、最大長を維持する。
- **戻る処理**: `history` の末尾要素を `pop` し、直前の要素を現在の `text` として復元して表示する。

### 4.6. クリア (`handleClear`)
- 現在の `text` を空文字列にする。
- 履歴（`history`）に空文字列の状態を追加する。

---

## 5. プレビューエリアの描画ロジック
- ソーステキスト内の独自記法 `<ruby=ルビ文字>対象文字</ruby>` を正規表現で抽出する。
- 【想定される正規表現】: `/<ruby=([^>]+)>([^<]+)<\/ruby>/g`
- 抽出した要素を、HTML標準の `<ruby>対象文字<rt>ルビ文字</rt></ruby>` に変換し、Reactのレンダーツリーに組み込んで表示する。
