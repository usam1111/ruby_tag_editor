# ルビタグ編集ツール 開発タスクリスト

設計書に基づいて、Reactアプリケーションを実装するためのタスクを細分化したリストです。進捗に応じてチェックを入れて管理します。

- [ ] React（Vite等）プロジェクトのセットアップ
- [ ] 不要な初期ファイルの整理
- [ ] 全体のレイアウト設計（CSS/スタイル定義）
  - [ ] 画面を左（ソース）、右（プレビュー）、下（ボタン）の3エリアに分割するスタイルを作成

## 2. コンポーネントの基本実装（UI・モック）

  - [ ] メインのレイアウト組み立て
- [ ] **SourceEditorコンポーネント**
  - [ ] `<textarea>` の記述と、タイトルの配置
- [ ] **Previewerコンポーネント**
  - [ ] テキスト表示用 `<div>` の記述と、タイトルの配置
- [ ] **ButtonGroupコンポーネント**
  - [ ] 3つのボタン（コピー、ひとつ戻る、クリア）の配置

## 3. 状態管理（State）の組み込み
- [ ] **Stateの定義 (in App.tsx / App.jsx)**
  - [ ] `text`: 現在のテキスト内容（初期値：「ここに<ruby=もじ>文字</ruby>をいれてください」）
  - [ ] `history`: 履歴保存用の配列（初期値：[初期テキスト]）
  - [ ] `isInitialText`: 初期テキストかどうかの判定フラグ（初期値：true）
- [ ] **Propsの伝搬**
  - [ ] 各子コンポーネントへのProps（状態と後述のハンドラ関数）の受け渡し

## 4. ソーステキストエリアの実装
- [ ] **直接入力の反映**
  - [ ] `<textarea>` の `onChange` イベント（テキスト変更のState反映）
- [ ] **初期テキストの消去機能**
  - [ ] `onMouseDown` イベントで、`isInitialText` が `true` の場合にテキストを空にする処理
  - [ ] 直打ち等でテキストが手動変更された際（`onChange`等）、確実に `isInitialText` を `false` に切り替える処理
- [ ] **ルビタグの自動挿入機能**
  - [ ] `onMouseUp` イベントの取得
  - [ ] 選択状態の判定（`selectionStart !== selectionEnd`）
  - [ ] 文字列を3分割（前半・選択部分・後半）して `<ruby=>` と `</ruby>` を結合するロジックの実装
  - [ ] `useRef` を使用して `<textarea>` のカーソル位置を `<ruby=` の直後（右側）に移動・選択解除する処理

## 5. ボタン機能の実装
- [ ] **ソースをコピー機能**
  - [ ] `navigator.clipboard.writeText` を用いたクリップボードへのコピー処理
- [ ] **履歴管理（ひとつ戻る）の実装**

  - [ ] 「ひとつ戻る」ボタン押下時の `history.pop()` と 前状態の復元処理
- [ ] **クリア機能**
  - [ ] テキストを空にし、履歴に反映させる処理

## 6. プレビューエリアの実装
